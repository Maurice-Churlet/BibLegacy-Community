# Parcours Evenement

## Avant l'evenement
- installer et verifier l'environnement de travail
- preparer un petit lot d'images representatif
- valider ou ajuster un profil de reference
- confirmer le workflow de sortie pour l'equipe

## Juste apres la course
- recuperer le lot photo source
- renseigner le dossier d'entree, le dossier de sortie et le prefixe
- choisir le profil adapte au contexte
- decider si le fallback ou TrOCR sont necessaires

## Pendant le traitement
- suivre la progression dans l'interface
- surveiller les logs utiles sans noyer l'operateur
- laisser le run aller a son terme ou l'interrompre proprement si besoin

## Apres la production
- ouvrir les rapports et verifier les indicateurs clefs
- controler les dossiers de sortie
- reprendre les cas difficiles dans l'outil web local
- reutiliser la page HTML et l'index photo pour la consultation ou l'integration WordPress par numero de dossard
- conserver les enseignements pour le prochain evenement

## Cycle d'amelioration
Sur plusieurs evenements, l'interet de BibLegacy augmente avec la capitalisation :
- profils plus pertinents
- meilleures decisions sur les options de renfort
- documentation et methodes plus stables
- moins de temps perdu en improvisation
